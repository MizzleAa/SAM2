### 1. 연구 배경 및 목적

SAM (Segment Anything Model)은 정적 이미지에서의 프롬프터블 세그멘테이션을 위한 파운데이션 모델로 큰 성공을 거두었으나, **영상(비디오) 도메인에서의 세그멘테이션 능력이 부재**했다. 실제 세계에서 시각적 세그먼트는 복잡한 움직임을 포함하며, AR/VR, 로보틱스, 자율 주행, 영상 편집 등 여러 응용 분야에서 시간적(temporal) 차원의 정확한 세그멘테이션이 필수적이다.

**SAM 2는 이미지와 비디오 모두에서 프롬프터블 세그멘테이션을 수행하는 통합 파운데이션 모델**을 목표로 한다.

---

### 2. 핵심 Task: Promptable Visual Segmentation (PVS)

| 항목 | 설명 |
|---|---|
| **입력 프롬프트** | 포인트(양/음성 클릭), 바운딩 박스, 마스크 |
| **프롬프트 위치** | 비디오의 **어떤 프레임**이든 가능 |
| **출력** | 시공간 마스크 (masklet) — 비디오 전체 프레임에 걸친 세그멘테이션 마스크 |
| **반복 보정** | 추가 프레임에 프롬프트를 제공하여 결과를 반복적으로 개선 가능 |
| **이미지 적용** | 이미지를 단일 프레임 비디오로 취급하여 동일한 모델로 처리 |

### 3. 모델 아키텍처

SAM 2는 **스트리밍 메모리 기반 트랜스포머 아키텍처**로 구성된다.

```
┌─────────────────────────────────────────────────────────────┐
│                    SAM 2 Architecture                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [Image Encoder] ──→ [Memory Attention] ──→ [Mask Decoder]  │
│       (Hiera)              ↑↓                   ↑           │
│                      [Memory Bank]        [Prompt Encoder]  │
│                            ↑                                │
│                     [Memory Encoder]                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1 Image Encoder
- **MAE 사전학습된 Hiera** 이미지 인코더 사용 (계층적 구조)
- 비디오 전체에서 **프레임당 1회만 실행** → 실시간 처리 가능
- 멀티스케일 특징 제공으로 디코딩 시 고해상도 정보 활용

#### 3.2 Memory Attention
- L개의 트랜스포머 블록 스택
- **Self-Attention** → **Cross-Attention** (과거 메모리 참조) → **MLP** 순서
- 현재 프레임 특징을 과거 프레임 특징/예측/프롬프트로 조건화
- 효율적 어텐션 커널 (FlashAttention 등) 활용 가능

#### 3.3 Prompt Encoder & Mask Decoder
- SAM과 동일한 프롬프트 인코더 (클릭, 박스, 마스크)
- **Two-way 트랜스포머 블록** 기반 디코더
- 모호한 프롬프트에 대해 **다중 마스크 예측** 지원
- **객체 존재 여부 예측 헤드** 추가 (가림/사라짐 처리)
- 계층적 이미지 인코더에서 **스킵 커넥션**으로 고해상도 임베딩 활용

#### 3.4 Memory Encoder & Memory Bank
- **Memory Encoder**: 출력 마스크를 다운샘플링 후 프레임 임베딩과 결합
- **Memory Bank**: FIFO 큐 구조
  - 최근 N개 프레임 메모리 (비프롬프트)
  - 최대 M개 프롬프트 프레임 메모리
  - **Object Pointers**: 경량 벡터로 고수준 의미 정보 저장
- 최근 프레임에는 **시간적 위치 정보** 임베딩, 프롬프트 프레임에는 미적용

#### 3.5 학습 방식
- 이미지 + 비디오 데이터로 **동시 학습**
- **인터랙티브 프롬프팅 시뮬레이션**: 8프레임 시퀀스, 최대 2프레임 프롬프트
- 초기 프롬프트 확률: GT 마스크(50%), 양성 클릭(25%), 바운딩 박스(25%)
- 학습 중 모델 예측 기반 보정 클릭 샘플링

### 4. 데이터 엔진 & SA-V 데이터셋

#### 4.1 데이터 엔진 (3단계)

| 단계 | 모델 | 프레임당 시간 | 편집 프레임 비율 | 프레임당 클릭 수 | Phase 1 대비 속도 |
|---|---|---|---|---|---|
| **Phase 1** | SAM 단독 | 37.8 s | 100% | 4.80 | 1× (기준) |
| **Phase 2** | SAM + SAM 2 Mask | 7.4 s | 23.25% | 3.61 | ~5.1× |
| **Phase 3** | SAM 2 (완전체) | 4.5 s | 19.04% | 2.68 | ~8.4× |

- **Phase 1**: SAM으로 프레임 단위 수동 어노테이션 (16K 마스클릿)
- **Phase 2**: SAM + SAM 2 Mask 조합, 시간적 전파 활용 (63.5K 마스클릿)
- **Phase 3**: SAM 2 완전체 사용, 메모리 기반 정제 클릭만으로 충분 (197.0K 마스클릿)
- **자동 마스클릿 생성(Auto)**: 정규 그리드 포인트로 후보 생성 → 검증 → 추가

#### 4.2 SA-V 데이터셋 규모

| 항목 | 수치 |
|---|---|
| 비디오 수 | **50.9K** |
| 총 마스클릿 수 | **642.6K** (수동 190.9K + 자동 451.7K) |
| 총 마스크 수 | **35.5M** (기존 최대 대비 **53배**) |
| 평균 비디오 길이 | 14초 |
| 장면 구성 | 실내 54%, 실외 46% |
| 사라짐/재등장 비율 | 42.5% |

기존 VOS 데이터셋과의 비교:

| 데이터셋 | 비디오 수 | 마스클릿 수 | 마스크 수 |
|---|---|---|---|
| DAVIS 2017 | 0.2K | 0.4K | 27.1K |
| YouTube-VOS | 4.5K | 8.6K | 197.3K |
| MOSE | 2.1K | 5.2K | 431.7K |
| BURST | 2.9K | 16.1K | 600.2K |
| **SA-V (Manual+Auto)** | **50.9K** | **642.6K** | **35.5M** |

### 5. 실험 결과

#### 5.1 프롬프터블 비디오 세그멘테이션 (Zero-shot)
- 9개 데이터셋에서 **offline/online 평가 모두 SAM+XMem++, SAM+Cutie 대비 우수**
- **기존 대비 3배 이상 적은 인터랙션**으로 더 높은 정확도 달성

#### 5.2 Semi-supervised VOS (17개 데이터셋 Zero-shot)

| 방법 | 1-click | 3-click | 5-click | bbox | GT mask |
|---|---|---|---|---|---|
| SAM+XMem++ | 56.9 | 68.4 | 70.6 | 67.6 | 72.7 |
| SAM+Cutie | 56.7 | 70.1 | 72.2 | 69.4 | 74.1 |
| **SAM 2** | **64.7** | **75.3** | **77.6** | **74.4** | **79.3** |

#### 5.3 이미지 세그멘테이션 (37개 Zero-shot 데이터셋)

| 모델 | 데이터 | SA-23 전체 (1/5-click mIoU) | FPS |
|---|---|---|---|
| SAM | SA-1B | 58.1 / 81.3 | 21.7 |
| SAM 2 | SA-1B | 58.9 / 81.7 | **130.1** |
| SAM 2 | our mix | **61.9 / 83.5** | **130.1** |

- SAM 대비 **정확도 향상 + 6배 빠른 속도**

#### 5.4 VOS 기존 SOTA 대비 성능 (J&F)

| 방법 | MOSE | DAVIS 2017 | LVOS | SA-V val | SA-V test | YT-VOS 2019 |
|---|---|---|---|---|---|---|
| Cutie-base+ | 71.7 | 88.1 | - | 61.3 | 62.8 | 87.5 |
| **SAM 2 (Hiera-B+)** | **76.6** | **90.2** | **78.0** | **76.8** | **77.0** | **88.6** |
| **SAM 2 (Hiera-L)** | **77.9** | **90.7** | **78.0** | **77.9** | **78.4** | **89.3** |

### 6. 핵심 모델 설계 결정 (Ablation 요약)

| 요소 | 최적 설정 | 핵심 발견 |
|---|---|---|
| 입력 해상도 | **1024×1024** | 해상도 증가 시 이미지/비디오 모두 향상 |
| 학습 프레임 수 | **8 프레임** | 비디오 벤치마크에서 큰 이득, 속도 균형 |
| 메모리 프레임 수 | **6개** | 시간적 컨텍스트와 연산 비용 균형 |
| 메모리 채널 수 | **64** | 적은 채널로도 성능 유지, 4배 메모리 절감 |
| 메모리 어텐션 | **(4 sa, 4 ca)** | 비디오 메트릭에서 향상 |
| 이미지 인코더 | **Hiera-B+ (기본) / Hiera-L (고정밀)** | 인코더 크기 증가 시 이미지/비디오 모두 향상 |
| 데이터 조합 | **VOS + SA-1B + SA-V + Internal** | 모든 데이터 조합이 최고 성능 |

### 7. 결론 및 시사점

1. **통합 아키텍처**: 이미지와 비디오를 하나의 모델로 처리 (이미지 = 단일 프레임 비디오)
2. **스트리밍 메모리**: 실시간 비디오 처리 가능한 메모리 기반 아키텍처
3. **데이터 엔진**: 모델-인-더-루프 방식으로 8.4배 빠른 어노테이션 달성
4. **SA-V 데이터셋**: 기존 대비 **53배** 더 큰 비디오 세그멘테이션 데이터셋
5. **성능**: 비디오에서 3배 적은 인터랙션으로 더 높은 정확도, 이미지에서 6배 빠른 속도
6. **오픈소스**: SA-V (CC BY 4.0), 모델 체크포인트, 학습 코드 (Apache 2.0) 공개
